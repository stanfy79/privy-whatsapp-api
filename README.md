## MoSCoW Framework - Future Prioritization

### Must Have
- Integrate the **Avalanche Fuji testnet** to simulate how the tool would behave in a production environment.
- Users should be able to send and receive **test transactions** on the MVP.

### Should Have
- Users should have an additional layer of authentication, such as a **unique six-digit PIN**, which must be entered before transactions are sent and executed.

### Could Have
- A **puzzle and game feature** to earn rewards as users interact with the tool.

### Won’t Have
- Users sending transactions to a **tag or an ENS (.eth) domain address** instead of typing out long addresses.
- A **privacy-layer integration**, such as the **E-ERC token standard**, that allows users to choose whether to send fully private transactions.

## 📖 Integrating BlockBot Widget | Overview

### BlockBot Widget UI Toolkit

**BlockBot Widget** is a collection of prebuilt UI interface components designed to let you embed a **secure, intuitive chat-driven crypto onboarding experience** on your website.

It can be fully customized to match your website’s design and helps you **engage and onboard new users effortlessly from across the globe**.

### BlockBot Widget Features

- **Arbitrum Sepolia** blockchain testnet transaction support.
- **Customizable design variants** — compact, QR code, and large.
- **Light or dark theme mode** support.
- Compatibility tested with **ReactJS, Next.js, TypeScript, HTML, and JavaScript**.
